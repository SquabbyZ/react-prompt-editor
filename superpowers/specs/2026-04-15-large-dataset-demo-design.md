# Large Dataset Demo Design

## 背景

当前 `PromptEditor` 文档已经覆盖了：

- 快速上手
- 基础工作流
- 预览模式
- 拖拽排序
- 流式优化
- 自定义优化流程
- 国际化与主题模式

但还缺少一个能直接体现“面向大规模节点”的文档示例。首页文案已经强调组件基于虚拟列表适合大 Prompt 树浏览与编辑，因此组件文档也需要补一个对应的可交互 demo，证明在大数据量下仍可正常加载、展开与滚动。

## 目标

新增一个独立的大数据性能示例，用最直接的方式展示 `PromptEditor` 在高节点规模下的使用体验。

该示例需要满足：

1. 用户能在 docs 中直接看到“大数据量”场景
2. 用户能切换不同数据量档位做体感对比
3. 最大档位使用总计约 `2000` 个节点的数据结构
4. 示例重点展示渲染、展开、滚动与基础编辑，不引入额外业务逻辑

## 非目标

- 不把这个示例做成完整性能测试面板
- 不在 docs 中输出复杂 benchmark 指标
- 不引入运行、AI 优化、拖拽等额外能力来干扰对“大数据展示”的理解
- 不为了该示例重写 `PromptEditor` 的整体架构

## 方案选择

本次采用“独立性能示例 + 预置数据量切换”的方案。

原因：

- 比把大数据模式塞进基础示例更清晰
- 比纯代码片段更能体现真实交互体验
- 比做复杂控制台更轻量，适合文档站长期维护

## 交互结构

示例页面只保留最必要的三部分：

### 1. 档位切换区

提供三档预置规模：

- `200 节点`
- `1000 节点`
- `2000 节点`

用户点击后直接替换当前编辑器数据，便于直观看不同规模下的渲染体感。

### 2. 数据摘要区

展示当前数据规模的简短摘要，例如：

- 根节点数量
- 总节点数量
- 子节点层级说明

该区域只用于帮助用户理解当前看到的数据结构，不展示复杂指标。

### 3. PromptEditor 展示区

使用单个 `PromptEditor` 承载大数据。

示例默认启用基础编辑体验，保留：

- 节点展开/折叠
- 内容编辑
- 预览滚动

不额外启用：

- `onRunRequest`
- `onOptimizeRequest`
- `draggable`

这样可以把注意力集中在大数据渲染本身。

## 数据结构设计

### 数据生成原则

示例数据不使用 `2000` 个平铺根节点，而采用“有限宽度 + 多层级”的树结构，模拟真实 Prompt 工程的组织方式。

### 推荐结构

最大档位采用：

- `200` 个一级节点
- 每个一级节点挂 `2` 个二级节点
- 每个二级节点继续挂 `2` 个三级节点

总量计算：

- 一级：`200`
- 二级：`400`
- 三级：`800`
- 四级：`600`

总计约：`2000`

实际生成时允许为了凑满总数做轻微裁切，但必须保证：

- 顶层节点数量稳定
- 结构有层级
- 节点标题和内容具备可读性

### 内容要求

每个节点至少包含：

- 稳定的标题序号
- 1 段简短 Markdown 内容

内容不需要很长，但要足够触发真实的节点高度计算逻辑，避免所有节点都过于扁平。

## 文档挂载位置

中文和英文组件文档都新增一个独立章节：

- 中文建议标题：`大数据示例`
- 英文建议标题：`Large Dataset Demo`

挂载位置放在：

- `拖拽排序` 之后
- `流式输出示例` 之前

原因是它更接近“组件容量与渲染能力展示”，比 AI 接入示例更基础。

## 实现边界

### 示例内实现

示例文件内部负责：

- 生成预置树数据
- 管理当前档位
- 维护 `PromptEditor` 的受控 `value`
- 展示简要统计信息

### 不在本次示例中实现

- 性能监控埋点
- FPS 展示
- 首屏耗时面板
- 自动压测
- 服务端数据拉取

## 风险与控制

### 风险 1：文档 demo 过重

控制方式：

- 使用档位切换而不是默认展示所有规模
- 保持 demo 控件最少
- 默认优先展示较小档位，避免首次打开压力过大

### 风险 2：大数据下出现列表错位

控制方式：

- 复用当前已修复的节点真实高度回写逻辑
- 在实现后重点验证展开、收起和滚动场景

### 风险 3：示例信息过杂

控制方式：

- 只展示“数据规模”和“结构层级”
- 不混入运行、优化、拖拽等能力

## 验证方式

实现完成后至少验证：

1. `200 / 1000 / 2000` 三档都能正常渲染
2. `2000` 节点场景下可正常滚动
3. 展开较大的一级节点后，下方节点位置正常，不出现大面积空白或重叠
4. 编辑任意节点内容后，列表高度更新正常
5. 中英文文档都能正常挂载该示例
6. docs build 通过

## 验收标准

- 文档中存在明确的“大数据示例”入口
- 用户能一眼看出组件支持高节点规模场景
- `2000` 节点示例可用且交互正常
- 示例结构清晰，不影响现有基础示例理解
- 中英文文档说明保持一致
