# 数据分析任务管理系统 - 功能使用说明

## 📖 产品概述

这是一个**智能化数据分析任务配置与管理平台**，用于帮助数据分析团队高效地组织、管理和跟踪复杂的数据分析项目。通过层级化的任务结构、依赖关系管理和可视化配置，让数据分析工作更加系统化和规范化。

---

## 🎯 核心功能

### 1. 任务层级管理

系统采用树形层级结构组织任务，支持最多 3 层嵌套：

- **Level 1**：一级任务（主分析方向）
  - 背景色：淡紫色渐变
  - 左边框：5px 紫色实线
  - 字体：20px 加粗
  
- **Level 2**：二级任务（子分析模块）
  - 背景色：淡绿色渐变
  - 左边框：4px 绿色实线
  - 字体：18px 半粗体
  - 缩进：30px
  
- **Level 3**：三级任务（具体分析项）
  - 背景色：淡粉色渐变
  - 左边框：3px 粉色实线
  - 字体：16px
  - 缩进：60px

### 2. 任务类型识别

系统自动识别三种任务类型，并通过右上角标签区分：

| 类型 | 标识 | 判断条件 | 颜色 |
|------|------|----------|------|
| 🔍 分析任务 | "分析" | 有 description 且 summary_node≠1 | 紫色渐变 |
| 📝 总结任务 | "总结" | summary_node=1 | 橙黄渐变 |
| 📌 普通任务 | "普通" | 其他情况 | 灰色 |

### 3. 任务属性配置

每个任务卡片包含以下可配置属性：

#### 3.1 基本信息
- **任务编号**：显示层级编号（如 1.2.1）
- **任务标题**：可编辑（仅未锁定的分析任务）
- **任务描述**：详细说明分析内容（可编辑）
- **总结要求**：针对总结任务的特殊要求（仅分析任务可编辑）

#### 3.2 数据模型
- 显示任务使用的数据模型/数据表
- 支持数组格式：`["table1", "table2"]`
- 也兼容对象格式（自动转换）

#### 3.3 依赖关系
- 显示前置依赖任务列表
- **智能依赖检查**：
  - 如果依赖的任务未锁定，显示橙色警告
  - 锁定任务时自动检查所有依赖是否已锁定
  - 解锁任务时自动解锁所有依赖它的任务

#### 3.4 关键词标签
- 以标签形式展示任务关键词
- 便于快速识别任务主题

## 🖱️ 交互操作详解

### 1. 数据导入

**操作路径**：点击顶部"📥 导入数据"按钮

**操作步骤**：
1. 点击按钮后弹出模态框
2. 在文本框中粘贴 JSON 格式的任务数据
3. 点击"确认导入"按钮
4. 系统自动解析并渲染任务列表

**数据格式示例**：
```json
[
  {
    "level": "1.1",
    "title": "用户访问模式分析",
    "description": "分析用户访问习惯...",
    "data_model": ["user_events", "sessions"],
    "keywords": ["DAU", "MAU"],
    "need2draw": true,
    "use_cache": false,
    "summary_node": 0,
    "depends_on": ["1"],
    "summary_requirements": "总结主要趋势..."
  }
]
```

**兼容性处理**：
- 自动添加缺失的 `need2draw` 和 `use_cache` 字段（默认 false）
- 自动将 `data_model` 对象格式转换为数组格式
- 自动初始化 `debugStatus` 为 'pending'

---

### 2. 任务编辑

#### 2.1 编辑标题
**触发条件**：分析任务且未锁定

**操作**：
- 直接点击标题输入框
- 修改后失去焦点自动保存
- 或按 Enter 键确认

**限制**：
- 锁定任务（use_cache=true）不可编辑
- 总结任务不可编辑

#### 2.2 编辑描述
**触发条件**：分析任务且未锁定

**操作**：
- 在描述文本域中编辑
- 支持多行文本
- 修改后自动保存

#### 2.3 编辑总结要求
**触发条件**：分析任务且未锁定

**显示规则**：
- 有内容时始终显示
- 编辑模式下即使为空也显示（方便添加）

**操作**：
- 在黄色背景的总结要求区域编辑
- 支持多行文本
- 可输入占位符提示

---

### 3. 任务锁定/解锁机制

#### 3.1 锁定任务（🔒）

**目的**：标记任务已完成调试，防止误修改

**操作**：
1. 点击任务右上角的解锁图标（🔓）
2. 系统检查：
   - ✅ 调试状态是否为"已完成"
   - ✅ 所有依赖任务是否已锁定
3. 检查通过 → 变为锁定状态（🔒）
4. 检查失败 → 弹出警告并高亮未锁定的依赖

**锁定后的变化**：
- 标题和描述变为只读
- "需要绘图"复选框禁用
- "调试"按钮禁用
- 锁定按钮变为红色锁定图标

**依赖检查逻辑**：
```
如果任务 A 依赖任务 B、C：
- B 和 C 都必须已锁定
- 否则无法锁定 A
- 系统会提示：无法锁定！以下依赖的任务尚未锁定：B, C
```

#### 3.2 解锁任务（🔓）

**操作**：
1. 点击任务右上角的锁定图标（🔒）
2. 任务变为未锁定状态
3. 调试状态重置为"pending"
4. 所有依赖此任务的子任务自动解锁

**级联解锁**：
```
如果解锁任务 A：
- 所有依赖 A 的任务 B、C、D 自动解锁
- B、C、D 的调试状态重置
```

---

### 4. 调试功能

**适用对象**：仅分析任务

**操作流程**：
1. 点击"⚡ 调试"按钮
2. 按钮显示"调试中..."
3. 模拟 300ms 延迟
4. 调试完成，按钮变为"✓ 已调试"（绿色）
5. 调试状态更新为 'completed'

**调试状态**：
- `pending`：未调试（红色按钮）
- `completed`：已调试（绿色按钮）

**调试与锁定的关系**：
- 必须先调试完成才能锁定任务
- 未调试时点击锁定会提示："请先完成调试后再锁定！"

---

### 5. 绘图标记

**适用对象**：仅分析任务

**操作**：
1. 勾选"需要绘图"复选框
2. 系统自动统计绘图任务数量
3. 底部仪表盘"需绘图"数字更新

**限制**：
- 锁定任务的绘图标记不可修改（已禁用）

**用途**：
- 标记需要可视化展示的任务
- 方便后续生成图表
- 统计需要绘图的任务总数

---

### 6. 数据导出

**操作路径**：点击顶部"📤 导出 JSON"按钮

**操作步骤**：
1. 点击按钮后弹出导出模态框
2. 系统自动清理临时属性（debugStatus）
3. 空字符串的 summary_requirements 自动删除
4. 显示格式化后的 JSON 数据

**导出选项**：

#### 6.1 复制到剪贴板
- 点击"📋 复制到剪贴板"按钮
- 自动选择全部文本
- 执行复制操作
- 弹出提示"已复制到剪贴板！"

#### 6.2 下载文件
- 点击"💾 下载文件"按钮
- 自动生成 JSON 文件
- 文件名格式：`task_config_{时间戳}.json`
- 浏览器自动下载

**导出数据示例**：
```json
[
  {
    "level": "1.1",
    "title": "用户访问模式分析",
    "description": "分析用户访问习惯...",
    "data_model": ["user_events", "sessions"],
    "keywords": ["DAU", "MAU"],
    "need2draw": true,
    "use_cache": true,
    "summary_node": 0,
    "depends_on": ["1"]
  }
]
```

---

### 7. 完成所有任务

**操作路径**：点击顶部"✅ 完成所有任务"按钮

**检查逻辑**：
1. 扫描所有分析任务
2. 检查是否有未锁定的任务
3. 如果有未锁定任务：
   - 弹出警告："还有 X 个任务未锁定！"
   - 自动高亮未完成任务（脉冲动画）
   - 持续 3 秒
4. 如果全部锁定：
   - 弹出成功提示："🎉 所有任务已完成！"

**高亮动画**：
- CSS 动画名称：pulse
- 效果：紫色光晕扩散
- 持续时间：2 秒循环，3 秒后消失

## 📊 数据仪表盘

页面底部固定显示实时统计数据：

```
┌─────────────┬─────────────┬─────────────┬─────────────┐
│  总任务数   │  分析任务   │   需绘图    │  已完成     │
│     20      │     15      │      8      │     12      │
└─────────────┴─────────────┴─────────────┴─────────────┘
```

**统计说明**：
- **总任务数**：所有任务数量
- **分析任务**：有 description 且 summary_node≠1 的任务
- **需绘图**：need2draw=true 的任务
- **已完成**：use_cache=true（已锁定）的任务

**更新机制**：
- 每次任务状态变化自动更新
- 包括：锁定/解锁、绘图标记切换、导入新数据

## 🎨 视觉交互细节

### 1. 悬停效果
- **任务卡片**：悬停时上浮 2px，阴影加深
- **按钮**：悬停时上浮 2px，显示阴影
- **锁定按钮**：悬停时放大 1.1 倍

### 2. 动画效果

#### 2.1 脉冲动画（pulse）
- **触发场景**：点击"完成所有任务"但有未完成项
- **效果**：紫色光晕从中心向外扩散
- **持续时间**：2 秒循环

#### 2.2 摇晃动画（shake）
- **触发场景**：锁定任务时依赖未锁定
- **效果**：左右摇晃 10px
- **持续时间**：0.5 秒

### 3. 颜色语义

| 颜色 | 含义 | 使用场景 |
|------|------|----------|
| 紫色 | 分析/编辑 | 分析任务、编辑状态 |
| 绿色 | 成功/安全 | 已锁定、已完成调试 |
| 红色 | 警告/错误 | 未锁定警告、调试按钮 |
| 橙色 | 提示/依赖 | 总结任务、依赖警告 |
| 灰色 | 普通/禁用 | 普通任务、禁用状态 |

## 🔄 典型工作流程

### 场景：完成一个数据分析项目

```
1. 导入项目任务配置
   ↓
2. 从 Level 1 开始，逐级编辑任务标题和描述
   ↓
3. 对每个分析任务：
   - 点击"⚡ 调试"进行调试
   - 调试完成后点击🔓→🔒锁定
   - 勾选"需要绘图"（如需要）
   ↓
4. 处理依赖关系：
   - 确保前置任务已锁定
   - 系统自动检查依赖完整性
   ↓
5. 完成所有任务后：
   - 点击"✅ 完成所有任务"
   - 系统验证所有任务已锁定
   ↓
6. 导出最终配置：
   - 点击"📤 导出 JSON"
   - 下载或复制配置文件
```

## ⚙️ 高级功能

### 1. 依赖链管理

**依赖传递**：
```
1 → 1.1 → 1.1.1
    → 1.2 → 1.2.1
```

**解锁连锁反应**：
```
解锁 1.1 → 自动解锁 1.1.1、1.1.2
```

**锁定前置检查**：
```
锁定 1.1.1 → 检查 1.1 是否锁定
          → 检查 1 是否锁定
```

### 2. 总结任务特殊处理

**总结任务特点**：
- 不可编辑标题和描述
- 没有"调试"和"绘图"选项
- 只有"总结要求"区域（如果有）
- 依赖多个前置分析任务

**总结要求来源**：
- 来自依赖的分析任务的 summary_requirements
- 或在分析任务编辑时填写

### 3. 数据模型兼容性

**支持的格式**：
```javascript
// 数组格式（推荐）
data_model: ["table1", "table2"]

// 对象格式（自动转换）
data_model: {
  "table1": {...},
  "table2": {...}
}
// 导入时自动转换为：["table1", "table2"]
```

## ❓ 常见问题

### Q1: 为什么无法锁定任务？
**A**: 可能原因：
1. 调试状态不是"已完成"
2. 依赖的任务未全部锁定
3. 系统会弹出具体原因提示

### Q2: 如何批量编辑任务？
**A**: 当前版本不支持批量编辑，需要逐个编辑。建议先规划好任务结构再导入。

### Q3: 导出后再导入会丢失数据吗？
**A**: 不会。导出时会清理临时状态（debugStatus），但核心数据完整保留。

### Q4: 如何修改已锁定的任务？
**A**: 点击锁定按钮（🔒）解锁 → 编辑内容 → 重新调试 → 再次锁定。

### Q5: 依赖关系如何影响工作流？
**A**: 依赖关系确保任务按正确顺序完成：
- 必须先完成前置任务
- 防止跳跃式工作
- 保证分析逻辑的连贯性
