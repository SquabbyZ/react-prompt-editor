# 包大小分析工具

本项目提供了包大小分析功能,帮助你了解构建产物的大小并进行优化。

## 使用方法

### 1. 运行分析命令

```bash
pnpm build:analyze
```

这个命令会:
1. 先执行完整的构建 (`pnpm build`)
2. 然后分析 `dist` 目录下的所有文件

### 2. 查看分析报告

分析完成后,你会看到类似以下的报告:

```
📦 包大小分析报告

构建目录: /path/to/project/dist

━━━ ESM 产物分析 ━━━

📄 esm/index.js
   原始大小:     370 Bytes | Gzip:    278 Bytes | Brotli:    213 Bytes
📄 esm/components/...
   ...

ESM 总计:
   原始大小:    252.68 KB | Gzip:     81.37 KB | Brotli:     68.24 KB

━━━ CJS 产物分析 ━━━
...

━━━ 样式文件分析 ━━━
...
```

## 报告说明

报告会显示三种格式的文件大小:

- **原始大小**: 文件的实际大小
- **Gzip**: 使用 gzip 压缩后的大小(大多数服务器支持)
- **Brotli**: 使用 brotli 压缩后的大小(更好的压缩率,现代浏览器支持)

## 优化建议

根据分析报告,你可以:

1. **检查大型依赖**: 查看哪些模块占用空间最大
2. **按需加载**: 考虑将大型模块改为按需加载
3. **Tree-shaking**: 确保未使用的代码被正确移除
4. **依赖去重**: 检查是否有重复的依赖包

## 技术实现

分析脚本位于 `scripts/analyze-bundle.js`,它会:
- 递归扫描 `dist` 目录
- 计算每个 JS/CSS 文件的原始大小
- 计算 gzip 和 brotli 压缩后的大小
- 按模块类型(ESM/CJS/Styles)分类展示

## 注意事项

- 分析的是构建后的产物,不是源代码
- ESM 产物通常比 CJS 大,因为保留了模块结构以支持 tree-shaking
- 实际使用时,打包工具会进一步处理和优化这些文件
