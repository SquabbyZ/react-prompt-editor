### 注意事项

⚠️ **Storybook 核心 UI 文本**

Storybook 10.x 版本的核心界面文本（如菜单、按钮、面板标题等）的完全汉化需要官方语言包支持。目前的配置提供了：

1. 语言切换基础设施
2. 文档区域的中文支持
3. 自定义翻译的框架

如需更完整的汉化，可以：
- 参考 Storybook 官方 i18n 文档
- 贡献中文语言包到 Storybook 官方
- 使用自定义 addons 实现更深层的汉化

## 🎨 编写中文故事

虽然故事文件名建议使用英文（避免兼容性问题），但你可以在故事内容中使用中文：

### 示例：Button.stories.ts

```typescript
import type { Meta, StoryObj } from '@storybook/react-vite'
import { Button } from './Button'

const meta = {
  title: '示例/按钮',  // ✅ 支持中文路径
  component: Button,
  parameters: {
    docs: {
      description: {
        component: `
## Button 按钮

按钮是常用的基础组件...
        `,  // ✅ 支持中文描述
      },
    },
  },
  argTypes: {
    label: {
      description: '按钮文本',  // ✅ 支持中文描述
    },
    primary: {
      description: '是否为主要样式',
    },
  },
} satisfies Meta<typeof Button>

export default meta
type Story = StoryObj<typeof meta>

// ✅ 支持中文故事名（但建议使用英文避免兼容性问题）
export const Primary: Story = {
  args: {
    primary: true,
    label: '主要按钮',  // ✅ 支持中文参数
  },
}
```

## 🔧 常见问题

### Q: 为什么有些界面还是英文？
A: Storybook 核心 UI 的完全汉化需要官方语言包支持。目前我们配置了语言切换框架和文档区域的中文支持。

### Q: 可以使用中文故事文件名吗？
A: 技术上可以，但可能导致兼容性问题。建议使用英文文件名，在故事内容中使用中文。

### Q: 如何自定义翻译？
A: 编辑 `.storybook/locale.ts` 文件，添加或修改翻译键值对。

### Q: 语言切换后没有立即生效？
A: 尝试刷新页面（Ctrl/Cmd + R）或重启 Storybook 服务器。
