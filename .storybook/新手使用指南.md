## 什么是 Storybook？

**Storybook 是一个 UI 组件开发环境**，可以帮助你：

- ✅ **独立开发组件** - 不需要运行整个应用
- ✅ **可视化测试** - 直观地查看组件的各种状态
- ✅ **自动生成文档** - 组件 API 文档自动生成
- ✅ **团队协作** - 设计师和产品经理可以查看组件库
- ✅ **完全汉化** - 界面已配置为中文，易于上手

### 为什么使用 Storybook？

**传统开发方式：**
```
运行整个应用 → 导航到某个页面 → 查看组件 → 修改代码 → 刷新页面
```

**使用 Storybook：**
```
打开 Storybook → 选择组件 → 查看状态 → 修改代码 → 自动刷新 ✨
```

## 核心概念

### 📖 什么是"故事"（Story）？

**故事 = 组件的一种状态或用法**

例如 Button 组件：
- `主要按钮` - 一个故事
- `次要按钮` - 另一个故事
- `禁用状态` - 又一个故事

### 📖 什么是"元数据"（Meta）？

**元数据 = 组件的配置信息**

包括：
- 组件在侧边栏的路径
- 组件的默认属性
- 文档说明
- 控件配置

### 📖 什么是"参数"（Args）？

**参数 = 传递给组件的 props**

```typescript
args: {
  label: '按钮',
  primary: true,
  size: 'large'
}
```

### 📖 什么是"控件"（Controls）？

**控件 = 可视化调整参数的工具**

在底部"参数"面板中，可以：
- 修改文本
- 选择颜色
- 切换布尔值
- 选择选项

## 编写第一个故事

### 步骤 1：创建组件

在 `src/components/` 创建你的组件：

```tsx
// src/components/Hello.tsx
interface HelloProps {
  name: string
}

export function Hello({ name }: HelloProps) {
  return (
    <div style={{ padding: '20px', borderRadius: '8px', background: '#f0f0f0' }}>
      <h2>你好，{name}！👋</h2>
      <p>欢迎使用 Storybook</p>
    </div>
  )
}
```

### 步骤 2：创建故事文件

在 `src/stories/` 创建故事文件：

```typescript
// src/stories/Hello.stories.ts
import type { Meta, StoryObj } from '@storybook/react-vite'
import { Hello } from '../components/Hello'

// 1. 配置元数据
const meta = {
  title: '我的组件/Hello',  // 在侧边栏显示的路径
  component: Hello,         // 组件
  parameters: {
    layout: 'centered',     // 居中显示
    docs: {
      description: {
        component: `
## Hello 组件

一个简单的问候组件，用于演示 Storybook 的基本用法。

### 特性
- ✅ 支持自定义名字
- ✅ 响应式设计
- ✅ 完全可访问
        `,
      },
    },
  },
  args: {
    name: '世界',  // 默认参数
  },
  argTypes: {
    name: {
      control: 'text',      // 文本输入控件
      description: '要问候的名字',
    },
  },
} satisfies Meta<typeof Hello>

export default meta
type Story = StoryObj<typeof meta>

// 2. 定义故事

/**
 * 默认故事
 * 使用默认参数显示组件
 */
export const 默认：Story = {}

/**
 * 自定义名字
 * 展示如何传递自定义名字
 */
export const 自定义名字：Story = {
  args: {
    name: 'Storybook',
  },
}

/**
 * 带表情符号
 * 展示支持 Unicode 字符
 */
export const 带表情符号：Story = {
  args: {
    name: '开发者 🚀',
  },
}
```

### 步骤 3：查看效果

1. 保存文件
2. Storybook 自动刷新
3. 在侧边栏找到"我的组件/Hello"
4. 点击故事查看效果

### 步骤 4：使用控件

1. 点击底部的"参数"面板
2. 修改"name"字段
3. 预览区域实时更新

```typescript
  },

  args: {
    label: 'Button',
  },
}
```

### 实战 2：创建表单输入组件

```tsx
// src/components/Input.tsx
interface InputProps {
  label: string
  placeholder?: string
  disabled?: boolean
  value?: string
  onChange?: (value: string) => void
}

export function Input({ 
  label, 
  placeholder = '请输入...',
  disabled = false,
  value = '',
  onChange 
}: InputProps) {
  return (
    <div style={{ marginBottom: '16px' }}>
      <label style={{ display: 'block', marginBottom: '8px', fontWeight: 'bold' }}>
        {label}
      </label>
      <input
        type="text"
        placeholder={placeholder}
        disabled={disabled}
        value={value}
        onChange={(e) => onChange?.(e.target.value)}
        style={{
          padding: '8px 12px',
          borderRadius: '4px',
          border: '1px solid #ccc',
          width: '100%',
          maxWidth: '300px',
        }}
      />
    </div>
  )
}
```

```typescript
// src/stories/Input.stories.ts
import type { Meta, StoryObj } from '@storybook/react-vite'

      control: 'text',
      description: '输入框的值',
    },
  },
} satisfies Meta<typeof Input>

export default meta
type Story = StoryObj<typeof meta>

/**
 * 默认状态
 * 普通的输入框
 */
export const 默认：Story = {
  args: {
    label: '用户名',

/**
```

## 最佳实践

### ✅ 推荐做法

#### 1. 为每种状态创建故事

```typescript
export const 默认：Story = {}
export const 禁用：Story = { args: { disabled: true } }
export const 加载中：Story = { args: { loading: true } }
export const 错误：Story = { args: { error: '错误信息' } }
```

#### 2. 使用有意义的故事名

```typescript
// ✅ 好的命名
export const 主要按钮：Story = {}
export const 带图标的按钮：Story = {}

// ❌ 避免的命名
export const Story1: Story = {}
export const Test: Story = {}
```

#### 3. 添加详细的文档

```typescript
parameters: {
  docs: {
    description: {
      component: `
## 组件说明

详细描述组件的用途、特性、使用场景等。

### 特性
- 特性 1
- 特性 2

### 使用场景
1. 场景 1
2. 场景 2
      `,
    },
  },
}
```

#### 4. 使用控件测试所有属性

```typescript
argTypes: {
  size: {
    control: { type: 'select' },
    options: ['small', 'medium', 'large'],
  },
  color: {
    control: 'color',
  },
}
```

### ❌ 避免的做法

#### 1. 不要在故事中写复杂逻辑

```typescript
// ❌ 不好的做法
export const Complex: Story = {
  args: {
    data: useMemo(() => heavyComputation(), []),
  },
}

// ✅ 好的做法
export const Simple: Story = {
  args: {
    data: ['简单', '数据'],
  },
}
```

#### 2. 不要依赖外部状态

```typescript
// ❌ 不好的做法
const { user } = useAuth()
export const Profile: Story = {
  args: { user },  // 依赖外部状态
}

// ✅ 好的做法
export const Profile: Story = {
  args: {
    user: { name: '张三', email: 'test@example.com' },
  },
}
```

#### 3. 不要忘记无障碍测试

```typescript
// 在底部面板点击"无障碍"
// 检查并修复所有问题
```

## 常见问题

### Q1: 为什么我的组件没有显示？

**检查清单：**
- ✅ 故事文件是否在 `src/stories/` 目录
- ✅ 文件名是否以 `.stories.ts` 或 `.stories.tsx` 结尾
- ✅ 是否正确导出了 `meta` 和故事
- ✅ `meta` 中是否指定了 `component`

**解决方法：**
```typescript
// ✅ 正确的结构
import type { Meta, StoryObj } from '@storybook/react-vite'
import { MyComponent } from './MyComponent'

const meta = {
  title: '我的组件/MyComponent',
  component: MyComponent,
} satisfies Meta<typeof MyComponent>

export default meta
type Story = StoryObj<typeof meta>

export const 默认：Story = {}
```

### Q2: 如何修改组件的默认值？

在 `meta.args` 中设置：

```typescript
const meta = {
  args: {
    label: '默认文本',
    size: 'medium',
  },
}
```

### Q3: 如何为特定故事设置不同的值？

在故事中覆盖 `args`：

```typescript
export const 自定义：Story = {
  args: {
    label: '自定义文本',
  },
}
```

### Q4: 如何使用装饰器？

装饰器用于包装组件，添加上下文：

```typescript
const meta = {
  decorators: [
    (Story) => (
      <div style={{ padding: '20px' }}>
        <Story />
      </div>
    ),
  ],
}
```

### Q5: 如何集成到我的项目？

Storybook 是独立的开发环境，不影响主应用：

1. 在 Storybook 中开发组件
2. 测试各种状态
3. 将组件复制到项目中
4. 在项目中导入使用

### Q6: 语言切换后没有立即生效？

**解决方法：**
1. 刷新页面（Cmd + R）
2. 清除浏览器缓存
3. 重启 Storybook：`Ctrl+C` 停止，然后重新运行 `npm run storybook`
